What is Baseline?

Baseline is a standard that indicates when web platform features become reliably available across major browsers.

  • ✅ Clear communication about feature support
  • ✅ Helps developers make informed decisions
  • ✅ Tracks features across Chrome, Firefox, Safari, Edge
  • ✅ Updated regularly by web standards organizations

The 3 Baseline States

  • Widely: Features that are fully supported across all modern browsers.
  • Newly: Features that have recently been implemented and may not be fully supported yet.
  • Limited: Features that are only available in specific browsers or under certain conditions.

✅ Widely Available

Safe to Use in Production

  • Available in Chrome, Firefox, Safari, Edge for 30+ months
  • Excellent cross-browser support
  • Can be used without fallbacks in most cases
  • Recommended for all projects

Examples: Flexbox, CSS Grid, Custom Properties, Calc(), CSS Filters

🟡 Newly Available

Use with Progressive Enhancement

  • Available in all major browsers for less than 30 months
  • Good modern browser support
  • Consider fallbacks for older browsers
  • Test thoroughly across browsers

Examples: Container Queries, :has(), @layer, @scope, Nesting, Popover API

❌ Limited Availability

Experimental - Use with Caution

  • Not available in all major browsers yet
  • May require feature flags or polyfills
  • Syntax may change
  • Best for experiments and prototypes
  • Plan for graceful degradation

Examples: Anchor Positioning, Scroll-driven Animations, View Transitions (cross-document)

Real-World Examples

✅ E-commerce Site (Wide Reach)

  • ✅ Use: Flexbox, Grid, Custom Props
  • 🟡 Consider: Container Queries, :has()
  • ❌ Avoid: Anchor Positioning

🎨 Design Tool (Modern Browsers)

  • ✅ Use: All Widely Available
  • 🟡 Use: All Newly Available
  • ⚠️ Try: Limited (with fallbacks)

📰 News Site (Accessibility Focus)

  • ✅ Use: Widely Available only
  • 🟡 Test: Newly Available carefully
  • ❌ Avoid: Limited features

🚀 Experimental Project

  • ✅ Use: Everything Widely/Newly
  • 🟡 Experiment: Limited features
  • ✅ Provide: Polyfills & fallbacks

Checking Baseline Status

  1. Visit web.dev/baseline
  2. Check MDN browser compatibility tables
  3. Look for Baseline badges in documentation
  4. Use @supports for feature detection
/* Feature detection example */
@supports (container-type: inline-size) {
  .card {
    container-type: inline-size;
  }
}

/* Fallback for browsers without support */
@supports not (container-type: inline-size) {
  .card {
    /* Use media queries instead */
  }
}

Key Takeaways

  • ✅ Widely Available = Safe for production (30+ months)
  • 🟡 Newly Available = Use with progressive enhancement (<30 months)
  • ❌ Limited Availability = Experimental, use with caution
  • 📊 Baseline helps you make informed decisions
  • 🎯 Choose features based on your project needs
  • 🔍 Always check browser support for your audience
  • ✨ Use @supports for feature detection
